## 核心模块说明

### 1. 认证与授权模块

#### 1.1 登录流程

**两阶段登录机制：**

1. **阶段一：身份认证** (`POST /login`)
   - 校验用户名和密码
   - 返回用户信息和所有角色列表
   - 不生成Token

2. **阶段二：确认角色** (`POST /login/confirm-role`)
   - 用户选择角色
   - 校验用户-角色关联关系
   - 生成JWT Token

**核心类：**
- `LoginController` - 登录控制器
- `UserService` / `UserServiceImpl` - 用户服务
- `JwtUtil` - JWT工具类

#### 1.2 JWT Token机制

**Token结构：**
```json
{
  "userId": 1,
  "roleId": 1,
  "iat": 1234567890,
  "exp": 1235173890
}
```

**配置：**
- Secret Key: `jwt.secret`（从配置文件读取）
- 过期时间: `jwt.expiration`（默认7天）

**核心方法：**
- `generateToken(userId, roleId)` - 生成Token
- `getUserIdFromToken(token)` - 获取用户ID
- `getRoleIdFromToken(token)` - 获取角色ID
- `validateToken(token)` - 校验Token有效性

### 2. 权限校验模块

#### 2.1 权限校验方式

**方式一：手动校验（当前使用）**
```java
@GetMapping("/users")
public ApiResponse<?> getUserList(@RequestHeader("Authorization") String token) {
    Integer userId = jwtUtil.getUserIdFromToken(extractToken(token));
    adminUtil.requireAdmin(userId, token);
    // ...
}
```

**方式二：注解方式（已实现，可选使用）**
```java
@RequireRole(requireAdmin = true)
@GetMapping("/users")
public ApiResponse<?> getUserList(...) {
    // ...
}
```

**核心类：**
- `AdminUtil` - 管理员权限校验工具
- `RoleCheckAspect` - 权限校验AOP切面
- `@RequireRole` - 权限校验注解

#### 2.2 细粒度权限校验

**评阅老师权限校验：**
- 通过`ThesisProcess.reviewerId`字段校验
- 只能为分配给自己的学生评分
- 实现类：`ReviewerThesisServiceImpl.validateReviewerAccess()`

**答辩小组权限校验：**
- 通过`ThesisProcess.defenseTeamId`字段校验
- 只能为分配给自己的学生评分
- 实现类：`DefenseTeamThesisServiceImpl.validateDefenseTeamAccess()`

### 3. 论文流程管理模块

#### 3.1 论文流程状态

**状态流转：**
```
INIT → TOPIC_SUBMITTED → TOPIC_APPROVED → 
OPENING_SUBMITTED → OPENING_APPROVED → 
MIDTERM_SUBMITTED → MIDTERM_APPROVED → 
FINAL_SUBMITTED → FINAL_APPROVED → 
DEFENSE_SCORED → COMPLETED
```

**核心实体：**
- `ThesisProcess` - 论文流程主表
- `MaterialHistory` - 材料历史版本表
- `ScoreSheet` - 评分表
- `FinalGrade` - 最终成绩表

#### 3.2 材料版本控制

**版本管理机制：**
- 每次上传新材料时，旧版本的`isLatest`设为`false`
- 新版本的`isLatest`设为`true`
- 版本号自动递增

**核心方法：**
- `uploadMaterial()` - 上传材料（自动创建新版本）
- `getMaterialHistory()` - 获取历史版本列表
- `previewHistoryMaterial()` - 预览历史版本
- `downloadHistoryMaterial()` - 下载历史版本

### 4. 评分系统模块

#### 4.1 评分结构

**指导老师评分（40%权重）：**
- 选题与文献综述：0-20分
- 创新性：0-10分
- 基础理论与专业知识：0-35分
- 态度、写作水平、规范、综合能力：0-35分
- **总分：100分**

**评阅老师评分（20%权重）：**
- 选题与文献综述：0-20分
- 创新性：0-10分
- 基础理论与专业知识：0-40分
- 写作水平与综合能力：0-30分
- **总分：100分**

**答辩小组评分（40%权重）：**
- 报告内容：0-40分
- 报告过程：0-10分
- 答辩表现：0-50分
- **总分：100分**

#### 4.2 最终成绩计算

**计算公式：**
```
最终成绩 = 指导老师评分 × 0.4 + 评阅老师评分 × 0.2 + 答辩小组评分 × 0.4
```

**成绩等级：**
- 90-100分：优秀（EXCELLENT）
- 80-89分：良好（GOOD）
- 70-79分：中等（AVERAGE）
- 60-69分：合格（PASS）
- 0-59分：不合格（FAIL）

**核心类：**
- `GradeService` / `GradeServiceImpl` - 成绩计算服务
- `tryCalculateFinalGrade()` - 自动计算最终成绩（当三方评分都齐全时）

### 5. 文件服务模块

#### 5.1 文件存储

**存储方式：**
- 当前使用本地文件系统
- 文件路径格式：`/upload/{year}/{month}/{day}/{filename}`
- 建议生产环境使用对象存储（OSS）

**核心类：**
- `FileService` / `FileServiceImpl` - 文件服务
- `uploadFile()` - 上传文件
- `downloadFile()` - 下载文件
- `previewFile()` - 预览文件
- `deleteFile()` - 删除文件

#### 5.2 文件类型支持

- PDF文档
- Word文档（.doc, .docx）
- 图片文件（.jpg, .png等）
- 其他常见文档格式

## 业务逻辑说明

### 1. 学生材料上传流程

```
1. 学生上传材料
   ↓
2. 创建MaterialHistory记录（version=1, isLatest=true）
   ↓
3. 更新ThesisProcess状态（如：INIT → TOPIC_SUBMITTED）
   ↓
4. 等待审核
```

### 2. 材料审核流程

```
1. 审核人（指导老师/专业负责人/学院领导）审核材料
   ↓
2. 如果通过：
   - 更新ThesisProcess状态
   - 更新审批字段（如：topicSupervisorApproved=true）
   ↓
3. 如果驳回：
   - 在MaterialHistory中记录rejectedReason
   - 学生可以查看驳回原因并重新上传
```

### 3. 评分流程

```
1. 指导老师提交评分
   ↓
2. 创建ScoreSheet记录（scorerRole=SUPERVISOR）
   ↓
3. 触发最终成绩计算（GradeService.tryCalculateFinalGrade）
   ↓
4. 检查是否三方评分都齐全
   ↓
5. 如果齐全：
   - 计算最终成绩（加权平均）
   - 确定成绩等级
   - 创建/更新FinalGrade记录
   - 更新ThesisProcess状态为COMPLETED
```

### 4. 归档流程

```
1. 管理员查询已完成论文（status=COMPLETED）
   ↓
2. 可以按学院、专业、学生姓名、论文标题筛选
   ↓
3. 可以批量导出材料（ZIP格式）
   ↓
4. 可以查看统计信息（按学院、专业统计）
```

## 安全机制

### 1. 密码安全

- **加密算法**：BCrypt（Spring Security提供）
- **加密强度**：自动生成salt，每次加密结果不同
- **存储方式**：数据库中只存储加密后的密码

### 2. JWT Token安全

- **签名算法**：HMAC-SHA256
- **Secret Key**：从配置文件读取（生产环境需修改）
- **过期时间**：7天（可配置）
- **Token验证**：每次请求都验证Token有效性和过期时间

### 3. 权限校验

**三层权限校验：**

1. **角色权限**：通过`UserRole`表关联用户和角色
2. **接口权限**：Controller层验证角色（管理员/学生/老师等）
3. **数据权限**：Service层验证数据归属（评阅老师只能为分配的学生评分）

### 4. 数据校验

- **DTO层校验**：使用`@Valid`、`@NotBlank`、`@NotNull`、`@Pattern`等注解
- **业务层校验**：Service层进行业务逻辑校验
- **全局异常处理**：统一处理校验失败的情况

## 异常处理

### 全局异常处理器

**`GlobalExceptionHandler`** 统一处理以下异常：

1. **BusinessException** - 业务异常
   - 状态码：400
   - 返回业务错误信息

2. **MethodArgumentNotValidException** - 参数校验异常
   - 状态码：400
   - 返回字段级错误信息

3. **BindException** - 参数绑定异常
   - 状态码：400
   - 返回字段级错误信息

4. **IllegalArgumentException** - 非法参数异常
   - 状态码：400
   - 返回错误信息

5. **Exception** - 其他未预期异常
   - 状态码：500
   - 记录错误日志
   - 返回通用错误信息

### 异常处理流程

```
Controller层
  ↓
Service层（抛出BusinessException）
  ↓
GlobalExceptionHandler（统一处理）
  ↓
返回统一格式的ApiResponse
```

- `isAdmin(userId, token)` - 判断是否为管理员
- `requireAdmin(userId, token)` - 要求管理员权限（否则抛异常）

**实现逻辑：**
1. 从Token中获取roleId
2. 查询Role表，检查roleCode是否为"ADMIN"
3. 或通过UserRole表查询用户是否拥有ADMIN角色

---

### 4. FileService

**文件服务接口**

**核心方法：**
- `uploadFile(file, processId, materialType, uploaderId)` - 上传文件
- `downloadFile(filePath)` - 下载文件
- `previewFile(filePath)` - 预览文件
- `deleteFile(filePath)` - 删除文件
- `getOriginalFileName(filePath)` - 获取原始文件名

**文件存储：**
- 本地文件系统
- 路径格式：`/upload/{year}/{month}/{day}/{filename}`

---

### 5. GradeService

**成绩计算服务**

**核心方法：**
- `tryCalculateFinalGrade(processId)` - 尝试计算最终成绩

**计算逻辑：**
1. 查询该流程的所有评分记录
2. 检查是否三方评分都齐全（指导老师、评阅老师、答辩小组）
3. 如果齐全，按权重计算最终成绩
4. 确定成绩等级
5. 保存到FinalGrade表
6. 更新ThesisProcess状态为COMPLETED

## 业务规则

### 1. 论文流程规则

- 每个学生只能有一个论文流程（`ThesisProcess`）
- 流程状态必须按顺序流转，不能跳过中间状态
- 只有当前状态的材料通过审核后，才能提交下一阶段材料

### 2. 材料上传规则

- 每次上传新材料时，自动创建新版本
- 旧版本的`isLatest`设为`false`
- 新版本的`isLatest`设为`true`
- 版本号自动递增

### 3. 评分规则

- 指导老师、评阅老师、答辩小组三方评分都齐全后，才计算最终成绩
- 最终成绩 = 指导老师评分 × 0.4 + 评阅老师评分 × 0.2 + 答辩小组评分 × 0.4
- 最终成绩确定后，论文流程状态自动更新为`COMPLETED`

### 4. 权限规则

- **学生**：只能操作自己的论文流程
- **指导老师**：只能操作分配给自己的学生
- **专业负责人**：可以查看和审核本专业的论文
- **学院领导**：可以查看和审核本学院的论文
- **评阅老师**：只能为分配给自己的学生评分（通过`reviewerId`校验）
- **答辩小组**：只能为分配给自己的学生评分（通过`defenseTeamId`校验）
- **管理员**：拥有所有权限

## 常见问题

### 1. Token过期

**问题**：Token过期后无法访问接口

**解决**：重新登录获取新Token

### 2. 权限不足

**问题**：访问接口时提示"无权限访问"

**解决**：
- 检查用户是否拥有相应角色
- 检查Token中的roleId是否正确
- 对于评阅老师和答辩小组，检查是否分配了相应的学生

### 3. 文件上传失败

**问题**：文件上传时提示失败

**解决**：
- 检查文件大小是否超过限制
- 检查文件格式是否支持
- 检查服务器磁盘空间

### 4. 数据校验失败

**问题**：提交数据时提示"参数校验失败"

**解决**：
- 检查必填字段是否都已填写
- 检查字段格式是否正确（如手机号、邮箱）
- 检查数值范围是否在允许范围内
